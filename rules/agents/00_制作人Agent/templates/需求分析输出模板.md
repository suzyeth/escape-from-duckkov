# 需求分析输出模板

> 使用说明：制作人Agent在Step 02中使用此模板输出分析结果

```
🎬 制作人 Agent 需求分析

┌─────────────────────────────────────────────────────┐
│  需求原文: {用户输入的原始需求}                       │
│  触发方式: {自然语言 / /gd:命令名}                   │
├─────────────────────────────────────────────────────┤
│  需求类型: {FEATURE/OPTIMIZE/BUGFIX/TEST/DOC/REVIEW/CONFIG/RESEARCH} │
│  类型置信度: {🟢高 / 🟡中 / 🔴低}                    │
│  判断来源: {命令直接指定 / 一级关键词命中 / 二级关键词+上下文 / 语义分析} │
│  需求名称: {简短的需求名称}                          │
│  需求规模: {中小(XS/S/M) / 大(L) / 超大(XL)}        │
│  流程模式: {标准模式 / 主从模式 / 深度模式}           │
│  判断依据: {为什么判断为这个类型和规模}               │
├─────────────────────────────────────────────────────┤
│  流转路径:                                          │
│  {根据类型显示对应的流转路径}                        │
│                                                     │
│  跳过阶段: {列出跳过的阶段，如无则显示"无"}          │
│  特殊环节: {L→主从架构 / XL→强制探测+架构评审+加强审查+测试策略 / 无} │
└─────────────────────────────────────────────────────┘

⚡ 流转至: {下一个Agent名称}
```

## 触发方式说明

| 触发方式 | 说明 |
|---------|------|
| 自然语言 | 用户用自然语言描述需求，制作人执行完整分析流程 |
| `/gd:命令名` | 用户使用快捷命令，命令已指定类型，制作人跳过类型判断但仍执行规模评估和复合检查 |

## 类型置信度说明

| 置信度 | 含义 | 判断来源 |
|--------|------|----------|
| 🟢 高 | 类型判断非常确定 | 命令直接指定 / 一级关键词精确命中 |
| 🟡 中 | 类型判断较为确定，但需上下文辅助 | 二级关键词 + 上下文线索确认 |
| 🔴 低 | 类型判断不确定，可能需要调整 | 纯语义分析，无明确关键词命中 |

> **命令覆盖说明**：当触发方式为 `/gd:` 命令且制作人判断命令指定的类型不准确时，输出框中额外显示：
> `⚠️ 命令指定为{原类型}，经分析实际为{新类型}，已自动调整`

## 判断来源说明

| 判断来源 | 含义 | 对应步骤 |
|---------|------|----------|
| 命令直接指定 | 用户通过 `/gd:` 命令直接指定了需求类型 | step-01 → 1.0 命令识别 |
| 一级关键词命中 | 需求文本中一级关键词直接命中 | step-01 → 1.3.1 |
| 二级关键词+上下文 | 二级关键词命中，结合上下文线索确认 | step-01 → 1.3.2 + 1.3.3 |
| 语义分析 | 无明确关键词，通过语义理解推断 | step-01 → 1.3.4 |
| 命令覆盖 | 命令指定了类型但制作人判断不准确，已覆盖 | step-01 → 1.0 + 1.3 |

> **📌 判断依据填写规范**：判断依据字段应具体说明为什么判断为该类型，而非笼统的"关键词命中"。
> - ✅ 好的示例：`一级关键词"报错"命中 + 上下文提到具体文件"ItemListPanel.cs" → BUGFIX`
> - ❌ 差的示例：`关键词命中`

## 流转路径速查

| 类型代码 | 流转路径 |
|----------|----------|
| `FEATURE` | 项目管理 → 策划 → 主程 → 程序 → QA |
| `OPTIMIZE` | 项目管理 → 主程 → 程序 → QA |
| `BUGFIX` | 项目管理 → 程序 → QA |
| `TEST` | 项目管理 → QA |
| `DOC` | 项目管理 → 主程 |
| `REVIEW` | 项目管理 → 主程 → 程序 |
| `CONFIG` | 项目管理 → 程序 → QA |
| `RESEARCH` | 策划（跳过项目管理） |

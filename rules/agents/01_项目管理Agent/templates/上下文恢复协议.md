# 🔄 上下文恢复协议

> 当需要在新会话中继续未完成的需求时，使用本协议快速恢复上下文。
> 本协议由项目管理Agent执行，其他Agent配合。

---

## 📋 恢复步骤

### 第1步：读取全局状态

读取 `.GameDev/_ProjectManagement/需求池.md`，找到 `status=🔄 进行中` 的需求。

**输出确认**：
```
📊 发现 X 个进行中的需求：
- REQ-001: XXX（当前在 程序Agent 编码阶段）
- REQ-002: XXX（当前在 策划Agent 策划阶段）
```

### 第2步：读取目标需求的状态追踪

读取目标需求的 `01_策划案.md`（或起始文档）头部的 frontmatter 状态追踪。

**关键信息提取**：
| 信息 | 读取字段 |
|------|----------|
| 当前负责Agent | `current_agent` |
| 当前步骤 | `current_step` |
| 已完成的步骤 | `steps_completed` |
| 进度百分比 | `progress` |
| 阻塞信息 | `blockers` |

### 第3步：加载对应Agent和步骤

1. 加载 `current_agent` 对应的 Agent 入口文件
2. 加载 `current_step` 对应的步骤文件
3. 读取上一步骤的产出物（如有）

### 第4步：输出恢复确认

```
🔄 上下文已恢复：
- 需求: REQ-XXX {需求名称}
- 类型: {需求类型}
- 当前Agent: {current_agent}
- 当前步骤: {current_step}/{total_steps}
- 进度: {progress}
- 上次更新: {last_updated}
- 已完成阶段: {已完成的Agent列表}

继续执行 {current_agent} 的步骤 {current_step}...
```

### 第5步：继续执行

从当前步骤继续执行，无需重新执行已完成的步骤。

---

## ⚠️ 恢复注意事项

| 注意事项 | 说明 |
|---------|------|
| **不要重做已完成步骤** | frontmatter 中 `status=done` 的步骤不需要重做 |
| **检查阻塞状态** | 如果 `blockers` 不为空，先确认阻塞是否已解除 |
| **读取产出物** | 恢复后先读取当前步骤的前置产出物，确认内容完整 |
| **更新时间戳** | 恢复后更新 `last_updated` 字段 |

---

## 🚀 快速恢复命令

用户可以说：
- `"继续上次的需求"` → 自动执行恢复协议
- `"恢复 REQ-003"` → 恢复指定需求
- `"查看当前进度"` → 读取需求池展示所有需求状态

# 需求状态追踪模板

> 使用说明：每个需求创建文档时，在 `01_策划案.md`（或对应的起始文档）头部添加此 frontmatter
> 项目管理Agent负责创建和维护，各Agent阶段完成后更新自己的步骤状态

```yaml
---
# 📊 需求状态追踪（由项目管理Agent维护）
req_id: REQ-XXX
req_name: "需求名称"
req_type: FEATURE           # FEATURE/FEATURE_UI/OPTIMIZE/BUGFIX/TEST/DOC/REVIEW/CONFIG/RESEARCH
complexity: 中等             # 简单/中等/复杂
scale: XS/S/M              # 📏 需求规模：XS/S/M(中小) / L(大) / XL(超大)
flow_mode: 标准模式          # 📏 流程模式：标准模式 / 主从模式 / 深度模式
status: in-progress         # waiting/in-progress/done/blocked/cancelled
current_agent: 程序Agent     # 当前负责的Agent
current_step: 3             # 当前步骤编号
total_steps: 7              # 总步骤数（根据需求类型自动计算）
progress: 43%               # 进度百分比（current_step / total_steps）

# 阶段流转记录
steps_completed:
  - agent: "制作人"
    step: "需求分类"
    status: done
    start_time: "2026-03-02 10:00"
    end_time: "2026-03-02 10:03"
    duration_min: 3
  - agent: "项目管理"
    step: "需求初始化"
    status: done
    start_time: "2026-03-02 10:03"
    end_time: "2026-03-02 10:08"
    duration_min: 5
  - agent: "策划"
    step: "策划案编写"
    status: done
    start_time: "2026-03-02 10:08"
    end_time: "2026-03-02 10:26"
    duration_min: 18
  - agent: "程序"
    step: "编码实现"
    status: in-progress
    start_time: "2026-03-02 10:48"
    end_time: null
    duration_min: null

# 时间追踪
created_at: "2026-03-02 10:00"
last_updated: "2026-03-02 11:15"
estimated_total_min: 120
actual_total_min: null        # 完成后填入

# 阻塞记录（如有）
blockers: []
# blockers:
#   - description: "依赖XXX模块未完成"
#     reported_at: "2026-03-02 11:00"
#     resolved_at: null
---
```

## 📝 使用说明

### 创建时机
- **项目管理Agent** 在 `step-01_需求初始化` 时创建
- 必须包含 `scale` 和 `flow_mode` 字段（由制作人Agent评估传入）

### 更新时机
| 时机 | 更新内容 | 操作者 |
|------|----------|--------|
| 阶段开始 | `current_agent`、`current_step`、新增`steps_completed`条目（status=in-progress） | 项目管理Agent |
| 阶段结束 | 对应条目的`end_time`、`duration_min`、`status=done` | 项目管理Agent |
| 遇到阻塞 | `status=blocked`、新增`blockers`条目 | 当前Agent |
| 阻塞解除 | `status=in-progress`、更新`blockers`的`resolved_at` | 当前Agent |
| 需求完成 | `status=done`、`actual_total_min`、`progress=100%` | 项目管理Agent |

### 跨会话恢复
当需要在新会话中继续未完成的需求时，只需读取此 frontmatter 即可快速恢复上下文。
详见 `上下文恢复协议.md`。

# 主从Agent架构（上下文管理）

> 版本: v1.0
> 最后更新: 2026-03-01
> 更新原因: 从CLAUDE.md拆分，独立管理

---

## 🎯 设计目的

当遇到复杂任务时，单个对话的上下文长度可能不够。通过主从架构：
- **主Agent** 负责任务拆分、结果汇总、全局把控
- **子Agent** 在独立上下文中执行子任务，只返回精简结果
- 有效节省上下文空间，支持更复杂的任务

---

## ⚡ 触发条件

> **📏 规模自适应联动**：当制作人评估需求规模为 **L(大)** 或 **XL(超大)** 时，自动触发主从架构。

| 条件 | 说明 |
|------|------|
| 制作人评估规模为 L 或 XL | **⭐ 首要触发条件**，由制作人Agent在需求分析时判断 |
| 复杂度为“复杂” | 项目管理评估为复杂需求 |
| 多模块功能 | 功能包含3个以上独立模块 |
| 代码量大 | 预估代码超过300行 |
| 上下文预警 | 检测到上下文使用超过60% |

## 📋 子任务执行流程

```
主Agent:
┌─────────────────────────────────────────────────────────────┐
│  1. 分析整体任务                                             │
│  2. 拆分为N个子任务                                          │
│  3. 为每个子任务创建任务卡片（保存到文件）                    │
│  4. 依次执行子任务（每个子任务是独立对话）                    │
│  5. 收集子任务结果（从文件读取）                              │
│  6. 汇总结果，继续下一步                                     │
└─────────────────────────────────────────────────────────────┘

子Agent:
┌─────────────────────────────────────────────────────────────┐
│  1. 读取任务卡片                                             │
│  2. 执行具体任务                                             │
│  3. 输出结果到文件                                           │
│  4. 返回精简摘要（不超过200字）                              │
└─────────────────────────────────────────────────────────────┘
```

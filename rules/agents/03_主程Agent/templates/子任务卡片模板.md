# 子任务卡片模板

> 使用说明：复杂任务拆分时，为每个子任务创建一份卡片
> 路径：`.GameDev/{功能名称}/_subtasks/task_XXX_input.md`

```markdown
# 子任务卡片

## 基本信息
| 属性 | 值 |
|------|-----|
| 任务ID | TASK-{序号} |
| 父需求 | REQ-XXX {需求名称} |
| 子任务名称 | {模块名称} |
| 执行者 | 程序 Agent |
| 优先级 | {1/2/3，1最高} |

## 任务描述
{具体要实现什么功能}

## 技术要求
- 命名空间: {namespace}
- 主要类: {类名列表}
- 需实现接口: {接口定义}

## 输入依赖
- 需要读取: {文件列表}
- 依赖的类: {类名列表}

## 输出要求
- 产出文件: {文件路径列表}
- 代码规范: 遵循项目编码规范

## 接口约定
{接口代码}

## 与其他模块的关系
- 被依赖: {哪些模块会调用本模块}
- 依赖: {本模块需要调用哪些模块}
```

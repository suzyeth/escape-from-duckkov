# Step M1: 任务拆分（主从模式）[MODE: DESIGN]

## 📍 当前位置
主从模式 | 触发条件：代码量>300行 或 独立模块≥3个 | 📐 设计模式

## 🎯 本步骤目标
将复杂任务拆分为独立子任务，创建任务卡片。

## 🔀 拆分原则
1. **高内聚低耦合**: 每个子任务是独立的模块
2. **明确边界**: 子任务之间的接口要清晰定义
3. **可独立测试**: 每个子任务的产出可以独立验证
4. **适当粒度**: 每个子任务代码量控制在100-200行

## 📋 执行清单

1. □ 分析功能模块，识别独立子任务
2. □ 为每个子任务创建卡片 `_subtasks/task_XXX_input.md`
   - 参考 `templates/子任务卡片模板.md`
3. □ 依次派发给程序Agent执行
4. □ 收集并汇总子任务结果

## 📝 汇总验证清单
- [ ] 所有接口已正确实现
- [ ] 模块间依赖关系正确
- [ ] 无循环依赖
- [ ] 命名规范统一

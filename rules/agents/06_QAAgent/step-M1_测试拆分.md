# Step M1: 测试拆分（主从模式）[MODE: DESIGN]

## 📍 当前位置
主从模式 | 触发条件：验收标准>10条 或 独立模块≥3个 | 📐 设计模式

## 🎯 本步骤目标
将复杂测试任务拆分为独立测试模块。

## 🔀 拆分原则
1. **模块独立**: 每个测试任务对应一个功能模块
2. **可独立执行**: 每个测试任务可以独立运行
3. **结果明确**: 每个测试任务有明确的通过/失败结果
4. **适当粒度**: 每个测试任务包含3-5个测试用例

## 📋 执行清单

1. □ 分析测试范围，识别独立测试模块
2. □ 为每个模块创建测试任务卡片
3. □ 依次执行各测试任务（或派发给子QA）
4. □ 汇总各模块测试结果

```markdown

{测试需要的前置条件}
```

## 📋 执行清单

1. □ **为每个验收标准编写测试用例**
   - 测试方法命名：`AC001_[测试名称]()`
   - 使用AAA模式：Arrange-Act-Assert
2. □ **编写边界条件测试**
3. □ **编写异常情况测试**
4. □ **编写PlayMode测试**（如需要MonoBehaviour生命周期）
5. □ 应用知识库中的最佳实践

## 📝 测试代码位置
`Assets/Tests/{功能名称}/Test_{功能名称}.cs`

参考 `templates/测试代码模板.md` 获取完整模板。

## 📝 测试编写规范

### 必须遵守
- 每个验收标准都有对应的测试用例
- 测试用例命名清晰，能看出测试内容
- Assert消息清晰，便于定位问题
- 测试之间相互独立，不依赖执行顺序
- SetUp/TearDown正确清理状态

### 测试分类标签
```csharp
[Category("验收标准")]  // AC测试
[Category("边界测试")]  // 边界条件
[Category("异常测试")]  // 异常情况
[Category("PlayMode")]  // PlayMode测试
```

# Step 03: 测试执行与报告 [MODE: VERIFY]

## 📍 当前位置
步骤 3/3 | 前置：测试编写已完成 | 后续：流转决策 | 🧪 验证模式

## 🎯 本步骤目标
执行自动化测试、生成报告、根据结果决定流转方向。

## 📋 执行清单

1. □ **执行所有测试用例**
2. □ **生成测试报告** `.GameDev/REQ-XXX/06_测试报告.md`
   - 参考 `templates/测试报告模板.md`
3. □ **分析失败用例**（如有）
4. □ **检查是否为已知问题**（对照失败模式库）
5. □ **做出流转决策**

## 🔀 流转决策

### 全部通过
```
1. 更新知识库（如有新发现）
2. ⚡ 流转至: 策划 Agent（交付）
```

### 存在失败
```
1. 生成Bug报告（参考 templates/Bug报告模板.md）
2. ⚡ 流转至: 程序 Agent（Bug修复）
3. 修复后重新执行失败的测试
4. 最多3轮修复，超过则请求用户介入
```

## 📝 Bug严重程度定义

| 级别 | 定义 | 处理方式 |
|------|------|----------|
| P0 阻塞 | 核心功能完全不可用 | 必须立即修复 |
| P1 严重 | 主要功能有缺陷 | 必须修复 |
| P2 一般 | 次要功能有问题 | 建议修复 |

## ⚠️ 本步骤常见陷阱

- 修复后只测修复点，不做回归测试
- 超过3轮仍不请求用户介入

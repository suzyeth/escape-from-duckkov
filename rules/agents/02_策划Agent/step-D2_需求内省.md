# Step D2: 需求内省 🪞 [MODE: REFLECT]

> **借鉴 SuperClaude `/sc:reflect` 机制**
> 本步骤在需求交付完成后**自动执行**，将经验从"勾选清单"升级为"结构化深度分析+自动归档"

## 📍 当前位置
需求内省 | 触发条件：Step D1 交付流程完成后自动进入

## 🎯 本步骤目标
对本次需求全流程进行结构化回顾，提炼成功/失败模式，将经验自动归档到知识库或规则文件。

## ⏱️ 时间约束
- **限时5分钟**，禁止过度分析拖慢交付
- 如果本次需求无明显问题且无返工，可执行**精简版**（只做第1-3步）

## 📋 执行清单

### 1. 📊 基础数据收集
- [ ] 从需求文档frontmatter中读取以下数据：
  - 需求ID、名称、类型、规模
  - 各阶段时长（预估 vs 实际）
  - 返工次数、Bug修复轮数
  - 质量门禁通过情况（一次通过 or 打回补充）
  - 对抗审查发现数（如有）

### 2. ✅ 成功模式识别（做得好的，值得复用）
- [ ] 回顾全流程，识别**做得好**的具体做法
- [ ] 每个成功模式需记录：
  - **做法描述**：具体做了什么
  - **效果**：带来了什么好的结果
  - **可复用性**：通用（适用其他项目）/ 项目特定
- [ ] 至少识别 **1个** 成功模式（如果确实没有亮点，记录"本次为常规执行，无特别成功模式"）

### 3. ❌ 失败模式识别（做得不好的，需要避免）
- [ ] 回顾全流程，识别**做得不好**的具体问题
- [ ] 每个失败模式需记录：
  - **问题描述**：具体出了什么问题
  - **根因**：简要5 Why分析（不需要完整5层，找到关键原因即可）
  - **影响**：造成了什么后果（返工、延时、质量问题等）
  - **改进方案**：具体要改什么（精确到文件路径）
- [ ] 特别关注：
  - 是否有被用户纠正的错误？（必须记录，这是最重要的失败模式）
  - 是否有质量门禁打回？（记录打回原因）
  - 是否有返工？（记录返工原因）

### 4. 💡 经验提炼
- [ ] 从成功/失败模式中提炼**可泛化的经验**
- [ ] 每条经验需标注：
  - **经验描述**：一句话总结
  - **适用范围**：通用 / 本项目
  - **证据**：来自哪个成功/失败模式

### 5. 📈 效率指标计算
- [ ] 计算以下指标（对比历史均值）：

| 指标 | 计算方式 | 目标 |
|------|---------|------|
| 时长准确率 | min(预估,实际) / max(预估,实际) × 100% | > 80% |
| 返工率 | 返工步骤数 / 总步骤数 × 100% | < 20% |
| 门禁一次通过率 | 一次通过的门禁数 / 经过的门禁总数 × 100% | > 70% |
| 对抗审查发现数 | 对抗审查发现的问题数 | ≥ 3 |

### 6. 📝 经验自动归档
- [ ] 按以下规则将经验归档到对应位置：

| 经验类型 | 归档位置 | 操作 |
|----------|---------|------|
| 通用成功模式 | 对应Agent的step文件或入口文件 | 追加到"最佳实践/常见陷阱"章节 |
| 通用失败模式 | 对应Agent的step文件或入口文件 | 追加到"常见陷阱"章节 |
| 项目特定经验 | `.GameDev/_Iteration/知识库_XXX.md` | 追加到对应职能知识库 |
| 🔴 影响正确性的经验 | **立即更新**对应规则文件 | 不等，当场改 |
| 🟡 影响效率的经验 | 记录到迭代记录，下次需求前更新 | 可延后 |
| 🟢 优化建议 | 记录到迭代记录 | 可选执行 |

### 7. 🎯 生成内省报告
- [ ] 使用 `templates/需求内省模板.md` 生成报告
- [ ] 报告保存位置：需求功能文档目录下（如 `.GameDev/{功能名称}/07_内省报告.md`）

## 📋 精简版执行清单（无返工、无明显问题时使用）

> 当本次需求满足以下条件时，可使用精简版：
> - 返工次数 = 0
> - Bug修复轮数 ≤ 1
> - 质量门禁全部一次通过
> - 未被用户纠正错误

精简版只执行：
1. ✅ 基础数据收集（第1步）
2. ✅ 成功模式识别（第2步）— 可简化为一句话
3. ✅ 效率指标计算（第5步）
4. ✅ 在内省报告中标注"精简版"

## ⚠️ 注意事项

- 🕐 **严格限时5分钟**：内省是为了改进，不是为了写报告。超时立即停止
- 📊 **数据驱动**：所有结论必须基于具体数据（时长、次数、门禁结果），禁止主观感受
- 🎯 **行动导向**：每个失败模式必须有"改进方案"且精确到文件路径，禁止空泛的"下次注意"
- 🔄 **与现有迭代机制互补**：本步骤是"需求级内省"，各Agent阶段流转前的反思清单是"阶段级反思"，两者互补不冲突
- 📋 **周期级内省由 `/gd:reflect` 命令手动触发**，不在此步骤中执行

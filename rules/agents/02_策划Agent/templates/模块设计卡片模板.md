# 模块设计卡片模板

> 使用说明：复杂需求拆分时，为每个模块创建一份卡片

```markdown
# 模块设计卡片

## 基本信息
| 属性 | 值 |
|------|-----|
| 模块ID | MOD-{序号} |
| 父需求 | REQ-XXX {需求名称} |
| 模块名称 | {模块名称} |

## 模块描述
{这个模块要实现什么}

## 核心机制
{模块的核心玩法/功能机制}

## 与其他模块的关系
- 依赖: {依赖哪些模块}
- 被依赖: {被哪些模块依赖}

## 验收标准
| ID | 验收标准 | 测试方法 |
|----|----------|----------|
| AC-001 | {标准描述} | {如何测试} |
```

```markdown
## 模块设计汇总

| 模块ID | 模块名称 | 状态 | 验收标准数 |

| MOD-002 | 逻辑模块 | ✅ | 6条 |

### 模块关联图
[描述模块间的关系]

### 整体验收标准
[汇总所有模块的验收标准]
```
